Accessibility

Web accessibility is an extremely important facet of web development. Making websites accessible to all people is imperative to helping all web users learn from and contribute to the web, providing new perspectives to the benefit of all. The WAI (Web Accessibility Initiative) is considered the main authority in setting web accessibility standards. An important component of the WAI is the WCAG (Web Content Accessibility Guidelines) which ensures that web content meets a certain standard of accessibility. The WCAG has four main criteria - ensuring that a website is Perceivable, Operable, Understandable, and Robust (He). Perceivability has to do with making sure that a website can be understood in text-based and audible formats without losing meaning in structure or content. Making text-based alternatives for images and adding closed-captions for video are examples of making a website perceivable. Operability is meant to ensure that users can navigate and operate a website using alternative methods. Providing proper, detailed headings and labels for content is an important element of making websites operable. Understandability is about making sure a website is understood by the user. It includes identifying the language of the site for translations, providing clear instructions for users, and providing guidance to prevent user errors on elements that might require user input. Finally, robustness is about making sure that any web technology is built in a way that is compatible with assistive technology programs (He). The web is increasingly important in our lives, from shopping, to employment, to gaming, to health care. By making sure that new web technologies are accessible, we can provide equal access to the web for people with disabilities and allow for more inclusive, active participation in the web and in society at large. As if that weren’t enough, the proper structuring and labelling that makes for good web accessibility is also good practice for search engine optimization as well! (Emmanuel)
