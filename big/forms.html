Forms

On the web, a form is what allows users to easily input data on a website, whether it be feedback, billing information, login information, UI preferences, or anything else. That data can then be sent to the server for storage and analysis, or it can update the user’s interface on the site. The features provided by a form are extremely useful for improving the user’s experience, and it can be incredibly valuable as a communication and data collection tool for an organization or company running a website. On the user end, forms can include text boxes, radio buttons, check boxes, dropdown lists, file uploaders, and more (Puri). As valuable of a tool as forms are, they can also be frustrating and off-putting to users if implemented incorrectly. In order to improve usability, it’s important to keep forms short and simple, only requesting data that you really need. It should also be clear to the user why you want this information, and what kind of data you expect in certain fields (Puri). Providing clear instructions to the user, and/or giving examples in front of the form will help provide a smooth and informed user experience. Ordering form fields by importance and allowing users to interact with the form using their keyboard are also considered to be general best practice when creating forms. Oftentimes we will want to control what kind of data is allowed in different fields, a process called form validation. If a user makes a mistake or inputs the wrong kind of data in a field, it is important that the site can identify for the user where the mistake was made, and what the user can do to correct it (Kavinda). Along with usability, visual layout and styling of form elements can make a form clearer, easier to use, and more appealing. By using CSS to style form elements we can change sizes, colors, shapes, borders, fonts, and more. Keeping the form as a single column will make the form easier to fill out. Indicating required fields with a clear visual marker will let users know what parts they need to fill out, and which parts of a form might be optional (Birkett). Making sure that our forms are visually simple, readable, and appealing will make it much more likely for users to fill them out.
